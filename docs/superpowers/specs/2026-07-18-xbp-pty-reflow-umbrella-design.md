## 1. Problem (observed in dogfood, build 15)

The phone renders desktop rows 1:1 on a canvas `cols × CELL_WIDTH` wide. Desktop terminals run ~110+ columns; the phone fits ~53 at 11pt mono. Result: every line runs ~2× past the screen edge, reading requires constant horizontal scrubbing, and paragraphs the desktop xterm soft-wrapped at desktop width are frozen at those column boundaries. Screenshots: `phone.PNG` vs `desktop.png` (2026-07-18).

## 2. Ratified decisions

1. **Reflow is the default rendering mode.** Desktop rows are rejoined into logical lines and re-wrapped at the phone's own column count. Vertical scroll only.
2. **"Desktop layout" is a toggle**, keeping today's 1:1 pan behavior verbatim as an escape hatch (TUI dialogs, tables, box art).
3. **Agent switcher becomes a dropdown.** The scrollable chips row is replaced by a single current-agent pill; tapping it opens an anchored menu listing all agents (ended agents included, dimmed). The header no longer scrolls.
4. **Layout verification moves to fixture-driven tests** so a TestFlight round-trip is not required to validate rendering changes. Fixtures are generated by 14all's real serializer from captured real PTY bytes.

## 3. Contract delta (v4 → v5)

One field, one version bump. `@ai-creed/command-contract` `0.1.0-alpha.3 → 0.1.0-alpha.4`, `COMMAND_CONTRACT_VERSION 4 → 5`.

| Item | Change |
| --- | --- |
| `PtyRow` | gains `wrapped?: boolean` — **true means this row is a soft-wrap continuation of the row at `line − 1`** (xterm's `isWrapped` semantics, i.e. the line began as an overflow of the previous line, not at an explicit newline). Absent or `false` = the row starts a new logical line. |
| Everything else | unchanged — capabilities, cursors, epochs, watermarks, refusal codes, `pty-changed` all stay v4 semantics. |

**Compatibility rules (normative):**

- Old host → new phone: `wrapped` absent → phone treats every row as a logical-line start. Reflow still wraps long rows at phone width; it just cannot rejoin desktop-wrapped paragraphs. Degraded, never wrong.
- New host → old phone: the old zod schema strips the unknown `wrapped` key on parse; v4 phones are unaffected.
- No handshake gating: the field is advisory rendering metadata, not a capability.

## 4. Reflow semantics (normative for the phone renderer)

Input: the store's `visibleRows` (sorted by `line`, one epoch) plus the phone's column budget `phoneCols = floor(panelInnerWidth / CELL_WIDTH)`.

1. **Join.** Row `r` is appended to the current logical line iff `r.wrapped === true` **and** `r.line` is exactly the previous visible row's `line + 1`. Any gap (trimmed rows, missing flag) starts a new logical line. If the first retained row carries `wrapped: true` (its head was trimmed away), it starts its own logical line. Joins concatenate `text` with no separator; style runs shift by the accumulated UTF-16 length.
2. **Wrap.** Each logical line is broken into display segments at `phoneCols`, measured by wcwidth (the same column arithmetic the contract's width invariant uses). Breaks fall on code-point boundaries; a combining mark never separates from its base; a wide glyph that would straddle the boundary moves wholly to the next segment.
3. **Slice.** Style runs are re-sliced across segment boundaries in UTF-16 units — same discipline as the existing per-row run slicing.
4. **Rule clip.** A logical line consisting solely of ≥8 repetitions of a single character from the set `─ ━ ═ ╌ ╍ ┄ ┅ ┈ ┉ - = _` (optionally padded by spaces) is **clipped** to `phoneCols` instead of wrapped, so horizontal rules read as full-width separators rather than two broken lines.
5. **Alt-screen.** When `altScreen: true`, the screen renders in desktop layout regardless of the toggle (positional grids are meaningless reflowed); the toggle is shown disabled. Returning to the normal screen restores the user's chosen mode.
6. **Desktop layout mode** renders exactly as shipped today: `pre` rows on a `cols × CELL_WIDTH` canvas, horizontal pan, right-edge fade.

Epoch, watermark, cursor, generation-guard, and follow-tail semantics are untouched — reflow is a pure view transform on `visibleRows`.

## 5. Switcher UX (normative composition; mockup is visual truth)

- Header order, all fixed (nothing scrolls): back chevron · identity block (spine, name, LIVE subtitle) · spacer · **current-agent pill** · **layout-toggle button**.
- Pill: `surface-2` fill, provider dot + provider display name + ▼ caret, 30pt tall with 44pt hit-slop. Rendered only when `list-ptys` returns >1 agent; single-agent sessions keep the provider in the subtitle and show no pill.
- Menu: anchored under the pill, radius-14 surface card, 44pt rows — provider dot + name, ✓ on the current agent, ended agents (`live: false`) dimmed with a mono `ended` tag but still selectable (their retained rows stay watchable). Scrim over the terminal; tap-out dismisses.
- **Selection paths.** A live agent runs the existing unsubscribe → subscribe → replay switch flow. An ended agent follows the **retained-rows path**: `subscribe-pty` refuses `no-live-agent` (the v1 umbrella's retention rule — rows stay pullable, subscribe does not), and the phone proceeds anyway with a full `pty-rows` replay (`cursor: null`), adopting `epoch`/`cols`/`watermark` from the pull snapshot instead of a subscribe ack; the ended banner renders and no hint subscription is active. If the follow-up `pty-rows` also refuses (`no-such-pty` — retention torn down), the ended empty state renders. The same rule applies when the screen opens directly onto an already-ended agent.
- Layout toggle: 30pt icon button, hairline outline when reflow (default), `surface-2` fill when desktop layout is active. Accessibility label "Desktop layout", state announced; pill label "Switch agent, current ⟨name⟩".

## 6. Fixture pipeline (test-without-TestFlight)

1. **Capture** (14all, dev-only): when `AI14ALL_PTY_CAPTURE_DIR` is set, the terminal service tees each session's raw PTY bytes to `<dir>/<sessionId>.bytes`. Off by default; never ships enabled.
2. **Generate** (14all): `scripts/generate-pty-fixture.ts --bytes <file> --cols <n> --rows <n> --out <file>` replays the bytes through the real `PtyMirror` + `serializePage` chain (cursor-chained until `more: false`) and emits `{ subscribe: { cols, epoch, watermark }, pages: PtyRowsPage[] }`.
3. **Consume** (xavier): the phone repo commits the generated JSON at `apps/phone/tests-render/fixtures/pty-real-session.json`; a smoke test replays it through the mocked session API into the real screen and asserts reflow layout, toggle parity, and style fidelity. Swapping in a newer capture is a file replacement, zero code change.

**Sequencing:** run the 14all SDD first (wrap flag + capture + fixture artifact), copy the fixture into ai-xavier, then run the xavier SDD. If the xavier SDD runs before a real capture exists, it lands with a script-generated fixture from a representative byte sample and the real capture replaces it later.

## 7. Joint acceptance (on device, operator-run)

- A live agent transcript is readable on the phone with no horizontal scrolling; paragraphs wrap at phone width with no mid-word desktop-column seams.
- Toggling desktop layout reproduces build-15 rendering exactly; toggling back restores reflow at the same scroll anchor (best effort).
- Dropdown switch works across live and ended agents; single-agent sessions show no pill.
- Alt-screen sessions auto-render desktop layout.
- Contract v5 phone against v4 desktop degrades per §3 (no crash, no misjoin).

Deferred items from the v1 ledger (FlatList virtualization depth, VoiceOver pass, real xterm palette swap, CELL_WIDTH device check) remain deferred and are not expanded by this feature.
