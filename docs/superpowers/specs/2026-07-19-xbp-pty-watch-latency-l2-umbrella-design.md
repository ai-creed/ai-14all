## 1. Where L2 stands

L2 added two optional contract primitives (contract v6, published) and a phone consumer that uses them: `PtyRowsArgs` gained `tail?` (newest-N entry) and `before?` (backward paging); `PtyRowsResult` gained `cursorBefore?`/`moreBefore?` (the backward channel). The phone opens the terminal by requesting `{ cursor: null, tail: N }` and lazily pulls `{ before }` pages on scroll-up.

Nothing honors those primitives yet. The real ai-14all host still runs the v5 serializer: `cursor: null` replays the whole epoch oldest-first, paged forward at 500 rows/page — up to ~20 sequential round-trips before the tail the user wants appears (`TERMINAL_SCROLLBACK_ROWS = 10_000`). Until the host serializes tail-first, the phone's fast-open path is exercised only by mocks and a v6 phone talking to the v5 host degrades to forward paging (contract compat, phone spec §Global constraints). This umbrella makes the host honor the primitives and proves the full loop with a fixture generated by the real serializer.

## 2. Ratified decisions

1. **Bounded backfill.** Backward paging serves only the retained scrollback (≤ `TERMINAL_SCROLLBACK_ROWS = 10_000` rows). `moreBefore` goes false once the caller's oldest row reaches `trimmedBefore` (the oldest retained line). No disk persistence, no cross-epoch history — exactly what contract v6 already assumes ("moreBefore is false once the oldest row is the oldest retained line").
2. **`moreBefore` is the v6 handshake.** Every `ok` page emits `moreBefore` as a **boolean** (never omitted); `cursorBefore` is present only when `moreBefore` is true. A v5 host omits both, so the phone's `moreBefore !== undefined` test on the entry page is exactly "am I talking to a v6 host?" (phone hook, `use-terminal-watch.ts` entry-page capture).
3. **Mode precedence: `before` → tail → forward.** Matches the phone's actual calls, which are mutually exclusive: entry sends `{ cursor: null, tail: N }`, scroll-up sends `{ cursor: null, before }`, live-tail sends `{ cursor }`.
4. **End-to-end via the shared fixture.** In addition to host unit tests, the fixture generator emits tail-first and backward pages so the phone's real consumer runs against real serializer output (extends the pty-reflow fixture pipeline, reflow umbrella §6).
5. **No `packages/xbp` reference-host change.** The `packages/xbp` reference host (`src/reference/host.ts`) is a generic schema-driven capability dispatcher: it validates a request against the capability schema and calls an injected `HostHandlers` function. It holds no PTY mirror, scrollback, or `serializePage` — v6's optional fields flow through it untouched. The only real serializer is ai-14all's `pty-serializer.ts`. (Recorded here so this is not mistaken for under-scoping: the phone spec's "reference host in `packages/xbp`" refers to that dispatcher, which serializes nothing.)

## 3. Contract (locked, v6 `alpha.5`)

Frozen; recorded for reference. No further contract motion in this arc.

Request `PtyRowsArgs`:

| Field | Semantics |
| --- | --- |
| `tail?: number` (positive int) | With `cursor: null`, return the **newest `tail` rows** instead of paging forward from the epoch start. Host clamps to its page cap. Ignored when `cursor !== null`. |
| `before?: string` | Fetch the page of rows **immediately older** than the caller's current oldest row, addressed by an opaque token from a prior response's `cursorBefore`. |

Response `PtyRowsResult` (ok arm):

| Field | Semantics |
| --- | --- |
| `cursorBefore?: string` | Opaque token addressing the page **older than this response's oldest row**. Present iff `moreBefore` is true. |
| `moreBefore?: boolean` | `false` once the oldest row is the oldest retained line. In `altScreen` always `false` (no scrollback). A v6 host always sends it; a v5 host omits it. |

**Consumption in ai-14all is via a vendored tarball, not the registry.** ai-14all pins `"@ai-creed/command-contract": "file:vendor/ai-creed-command-contract-0.1.0-alpha.4.tgz"`. The GitHub Packages publish does not change what ai-14all builds against — the child spec re-vendors an `alpha.5` tarball and bumps the `file:` refs (child §1).

## 4. Serializer semantics (normative — the behavior the real host and any conformance fixture must satisfy)

Absolute-line model (unchanged from the shipped serializer): the retained window is `[first .. last]` where `first = trimmedBefore` and `last = trimmedBefore + buffer.length − 1`. An empty buffer has `last = first − 1` (no rows). `serializeRow` is unchanged.

**Mode dispatch** (evaluate in order):

1. **`before` present → Backward.**
2. **`cursor === null` and `tail` present → Tail-first.**
3. **Otherwise → Forward** (today's `serializePage` selection: fresh full snapshot on `cursor === null`, else the `(stamp, line)` live-delta, capped).

**Tail-first.** Let `n = min(tail, cap)`. Emit lines `[max(first, last − n + 1) .. last]` (the newest `n` retained rows, contiguous ascending). Forward `cursor = encode({ epoch, watermark: mirror.watermark, line: last })` — positions the phone to resume live-tail from *now*, not to re-replay. `more = false` (the tail already ends at the newest row; new output arrives via live-tail).

**Backward.** Decode `before`. If it is unreadable, or its epoch ≠ `mirror.epoch`, or its boundary line `B ≤ first`, return an **empty older page** (`rows: []`, `moreBefore: false`, no `cursorBefore`) stamped at the current epoch — never an error. Otherwise emit lines `[max(first, B − cap) .. B − 1]` (up to `cap` rows immediately older than `B`, contiguous ascending). The forward `cursor`/`more` on a backward page are **ignored by the phone** (`use-terminal-watch.ts` `backfill()` consumes only `rows`, `epoch`, `cols`, `altScreen`, `cursorBefore`, `moreBefore`); emit the live-tail resume cursor and `more = false` to satisfy the non-null `cursor` contract.

**Uniform backward channel** (applied to the emitted rows of every mode):

- `altScreen` ⇒ `moreBefore = false`, `cursorBefore` omitted (the alternate buffer has no scrollback).
- Else if no rows ⇒ `moreBefore = false`, `cursorBefore` omitted.
- Else let `oldest = min(row.line)` over the emitted rows (= `rows[0].line` for the contiguous tail/backward pages). `moreBefore = oldest > first`; `cursorBefore = moreBefore ? encode({ epoch, watermark: 0, line: oldest }) : undefined`.

The backward token reuses the existing `(epoch, watermark, line)` cursor codec (`pty-cursor.ts`) — no new token type; `watermark` is unused on the backward axis and set to 0.

**Epoch/watermark/trim untouched.** Tail and backward are pure read shapes over the same retained buffer; they do not tick, stamp, resize, or trim.

## 5. Shared fixture shape (cross-repo contract)

The pty-reflow fixture artifact is `{ subscribe: { cols, epoch, watermark }, pages: PtyRowsPage[] }`, validated by `scripts/pty-fixture-schema.ts` (which re-adds the `{ ok: true }` wire envelope before parsing against the vendored contract). L2 extends the artifact **additively**:

```jsonc
{
  "subscribe": { "cols": 80, "epoch": 1, "watermark": 12 },
  "pages":        [ /* forward chain, cursor:null → more:false — unchanged */ ],
  "tailPage":     { /* one PtyRowsPage from { cursor:null, tail:N } */ },
  "backwardPages":[ /* chain: follow tailPage.cursorBefore via { before } until moreBefore:false */ ]
}
```

- `tailPage` and `backwardPages` carry the L2 fields (`cursorBefore`/`moreBefore`); the artifact schema extension (child §4) validates them through the same envelope.
- Old fixtures without the new keys stay valid (the reflow smoke test still reads `pages`); the phone smoke test reads the new keys when present.
- **Generator (ai-14all)** produces the artifact; **consumer (ai-xavier)** commits a byte-identical copy at `apps/phone/tests-render/fixtures/pty-real-session.json`. File copy is the only handoff.

## 7. Joint acceptance (on device, operator-run, after both repos land)

- A long, uncached session opens on the newest screen in a **single** `pty-rows` round trip (tail-first), not after a visible oldest-first staircase.
- Scrolling up lazily backfills older rows in order, with no scroll jump, until the start of retained history; the footer then reports start-of-history and stops requesting.
- A v6 phone against the now-v6 host performs no full forward replay on entry; against a still-v5 host it degrades to forward paging with no backfill affordance (contract compat).
- `altScreen` sessions issue no backward pulls and show no backfill footer (phone acceptance guard); entering alt mid-session suppresses the affordance, exiting restores it.
- The shared fixture's `tailPage`/`backwardPages`, generated by the real serializer, drive the real phone consumer green in CI without a TestFlight round-trip.

Deferred (not this arc): disk-persistent history beyond the 10k scrollback; the visible epoch/seam divider and "Show earlier" affordance; replacing the synthesized fixture with a real captured session (`mem-2026-07-18` parked item).
